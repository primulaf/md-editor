# 侧栏空间优化实施计划

日期：2026-07-15
依据：`docs/specs/2026-07-15-sidebar-space-optimization.md`
状态：已实施并完成回归验证

## 实施步骤

### 1. 精简页面结构

- 从 `index.html` 删除整个 `.tips` 功能介绍区域。
- 将统计面板标题从“统计信息”缩短为“统计”。
- 保留四个统计值元素的 `id`，避免修改 `app.js` 数据更新逻辑。

### 2. 重排侧栏空间

- 让侧栏自身隐藏纵向溢出，目录列表继续作为唯一的内部滚动区域。
- 将 `.toc-panel` 改为占据全部剩余空间，并允许在低高度下收缩。
- 保持品牌和工具栏为固定高度，统计摘要固定在侧栏底部。

### 3. 压缩统计摘要

- 将两列卡片网格改成四列单排。
- 移除统计项背景、外框和大内边距。
- 数值置于标签上方，用轻量竖分隔线区分项目。
- 使用受限字号，兼顾五档缩放和最长标签宽度。

### 4. 清理废弃样式

- 从 `style.css` 删除 `.tips` 相关选择器。
- 将面板公共样式限制为目录和统计，避免保留无效规则。

### 5. 验证

- 运行 `npm test` 和 `git diff --check`。
- 在 1280 × 720、超大字号下测量目录和统计面板高度。
- 在 800 × 900 下检查横向溢出、按钮和统计文字。
- 回归目录折叠、标题跳转和高亮跟随。

## 完成标准

- 功能介绍区域不存在。
- 统计面板达到 Spec 高度目标，四项数据仍正常更新。
- 1280 × 720、超大字号时目录列表可视高度不少于 360px。
- 桌面和窄屏均无文字溢出或区域重叠。

## 验证结果

- 1280 × 720、超大字号：目录列表可视高度 389px，统计面板高度 75px。
- 桌面侧栏无整体纵向溢出，功能介绍节点数量为 0。
- 800 × 900：无页面横向溢出，工具按钮和统计项均无文字溢出。
- 窄屏统计区可通过侧栏滚动访问。
- 目录折叠、三级标题跳转和高亮跟随通过，浏览器控制台无警告或错误。
- `npm test` 与 `git diff --check` 通过。
