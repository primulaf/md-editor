# v1.4.0 迭代实施计划

日期：2026-07-15
依据：`docs/specs/2026-07-15-ux-iteration-1.4.0.md`
状态：已实施并完成回归验证

## 目标

在不改变 Markdown 渲染核心和文件安全模型的前提下，完成侧栏工具区压缩、目录树折叠、五档字号和动态标签页标题，并保持现有目录跳转、同步滚动、文件载入及未保存提醒稳定。

## 实施步骤

### 1. 统一文档名称与未保存状态

- 增加统一的文档标题刷新函数，同时更新编辑区文件名和浏览器标签页标题。
- 标签页采用 `文件名 - md.` 格式，未保存时增加 `* ` 前缀。
- 覆盖缓存恢复、文件选择、文件关联、拖拽打开、新建、保存和编辑状态变化。

### 2. 扩展字号系统

- 将字号档位从三档扩展为 `small / medium / large / xlarge / xxlarge`。
- 保持旧的本地存储值兼容，非法值自动回退到小号。
- 调整 CSS 字号变量，确保侧栏、统计、目录、编辑区和预览区同步缩放。

### 3. 压缩侧栏工具区

- 将五个纵向按钮改为三列、两行的紧凑网格。
- “字号”占第二行两列，“新建”占一列。
- 缩小按钮高度、侧栏间距和工具区占用，同时保留文字、焦点样式和足够点击面积。

### 4. 重构目录为可折叠树

- 将渲染后的标题转换为树结构，保留标题层级、锚点和父子关系。
- 有子项的节点显示折叠按钮；一级默认展开，二级及以下有子项时默认折叠。
- 用户点击标题仍跳转；点击折叠按钮只展开或收起分支。
- 当前高亮标题位于折叠分支时自动展开父级，并滚入目录可视区域。
- 文档切换时重置折叠状态，文档内部重渲染时尽量保留用户选择。

### 5. 回归与发布元数据

- 验证空白文件、重复标题、跳级标题和长文档。
- 验证目录点击后编辑区和预览区均跳转，滚动时目录高亮和自动跟随正常。
- 验证五档字号刷新后保持，超大字号下控件不溢出。
- 更新 README、Manifest 和 npm 包版本到 `1.4.0`。

## 完成标准

- Spec 中四项建议全部可用。
- JavaScript 通过语法检查，Manifest 可解析。
- 桌面与窄屏布局无文字重叠或不可操作控件。
- 不覆盖当前工作区中与本次迭代无关的文件变更。

## 验证结果

- `npm test` 通过，Manifest 与 npm 元数据可正常解析。
- 五档字号循环及本地偏好恢复通过。
- 目录默认折叠、手动展开、双区跳转、高亮跟随和父级自动展开通过。
- 只修改正文时，目录折叠状态与标题锚点保持稳定。
- 空白文档和重复标题场景通过。
- 桌面端与 800px 窄屏布局无横向溢出；超大字号下工具按钮无文字溢出。
