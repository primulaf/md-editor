# 侧栏空间优化 Spec

日期：2026-07-15
目标版本：v1.4.0 体验修订
范围：Chrome 扩展 `md.` 的侧栏垂直空间分配。

## 背景

可折叠目录上线后，侧栏的信息优先级发生了变化：目录已经成为侧栏中最重要、最高频的区域，但当前统计信息仍使用四张卡片，功能介绍又长期占据底部空间。在 720px 高度和超大字号下，目录自身的可滚动区域只有约 91px，无法满足长文档定位需求。

## 目标

- 让目录区域占据侧栏除品牌、工具栏和紧凑统计外的全部剩余高度。
- 显著压缩统计信息，但保留总字数、行数、标题数和阅读时间四项数据。
- 删除侧栏功能介绍，避免重复展示用户已经熟悉的能力。
- 保持现有 GitHub 风格、五档字号、目录折叠和统计逻辑不变。

## 非目标

- 不调整侧栏宽度。
- 不改变目录树层级、跳转、高亮和折叠逻辑。
- 不增加统计信息折叠按钮或新的设置项。
- 不调整编辑区和预览区的布局比例。

## 信息层级

侧栏从上到下固定为：

1. 品牌。
2. 文档操作工具栏。
3. 目录，占据全部弹性空间。
4. 紧凑统计摘要。

功能介绍区域从 DOM 和 CSS 中完全删除。

## 目录区域

- `.toc-panel` 使用 `flex: 1` 占据剩余高度。
- 移除当前 `150px` 的固定最小高度约束，允许小视口下由整体布局合理收缩。
- 目录列表自身保持独立滚动，不让长目录推动整个侧栏滚动。
- 桌面端侧栏不应出现由内部内容造成的整体纵向滚动条。

## 统计摘要

### 布局

- 标题由“统计信息”缩短为“统计”。
- 四项数据改为单排四列：`总字数 / 行数 / 标题数 / 阅读时间`。
- 移除每项独立的背景、边框和卡片内边距。
- 使用轻量竖分隔线区分四项数据。
- 数值在上，标签在下，保持快速扫读。

### 字号

- 标签继续作为功能性小字，控制在 11–13px。
- 数值控制在 13–16px，并继续响应五档字号。
- “阅读时间”等较长标签在超大字号下不能溢出或换成不可读的多行。

### 高度

- 小号字体时统计面板总高度目标不超过 72px。
- 超大字号时统计面板总高度目标不超过 86px。

## 响应式要求

- 1280 × 720、侧栏打开、超大字号时，目录列表可视高度不少于 360px。
- 800 × 900 窄屏时，侧栏无横向溢出；工具栏、目录和统计仍可访问。
- 任意字号下统计项文字不得互相重叠。

## 验收标准

- 功能介绍区域完全消失。
- 四项统计数据仍实时更新且语义不变。
- 目录占据侧栏主要空间，并保留独立滚动。
- 目录折叠、点击跳转、高亮跟随和自动滚入可视区域无回归。
- 桌面与窄屏视觉检查通过，无文字溢出和布局重叠。
