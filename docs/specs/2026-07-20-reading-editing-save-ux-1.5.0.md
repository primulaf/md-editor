# v1.5.0 阅读与保存体验重构 Spec

## 目标

- 打开文件后默认进入阅读模式，只显示目录与预览区。
- 用户主动进入编辑模式后才显示编辑区和保存操作。
- 将“保存”“另存为”“导出 HTML”定义为三个互不混淆的命令。
- 在 Chrome 允许的范围内写回源文件，并准确说明没有写入权限的状态。
- 文档状态按标签页隔离，避免多个文件之间互相污染。

## 文档状态

界面状态：

- `view`：阅读模式，预览区占满内容区域。
- `edit`：编辑模式，编辑区与预览区可分栏或单独显示。

来源状态：

- `new`：新建文档，尚无磁盘目标。
- `linked`：持有可写文件句柄，可直接保存。
- `unlinked`：通过文件关联、拖拽或兼容文件选择器读取，不能直接写回。

修改状态：

- `clean`：内容与最近一次成功落盘或下载的版本一致。
- `dirty`：存在尚未持久化的修改。

## 交互

### 阅读模式

- 默认隐藏编辑区及其折叠占位，预览区使用全部可用宽度。
- 预览区标题栏提供“编辑文档”按钮。
- 侧栏提供打开、导出 HTML、字号和新建。
- 保存与另存为只在编辑模式出现。

### 编辑模式

- 显示编辑区并恢复上次保存的分栏比例。
- 预览区标题栏提供“完成编辑”，编辑区关闭按钮行为一致。
- “完成编辑”不丢弃修改，也不清除未保存状态。
- 新建文档后直接进入编辑模式。

### 保存

- `linked`：检查外部修改后写回当前文件。
- `new`：打开系统“另存为”选择器，成功后转为 `linked`。
- `unlinked`：显示自定义说明，允许选择源文件、另存为或取消。
- 不支持 File System Access API 时降级为下载副本，并明确提示源文件未更改。
- 仅在写入流成功关闭后清除 dirty 状态。

### 另存为

- 始终创建或选择一个新的 Markdown 文件。
- 成功后新文件成为当前保存目标，文件名和标签页标题同步更新。

### 导出 HTML

- 只生成 HTML，不改变 Markdown 文件句柄、文件名或 dirty 状态。

## 数据安全

- 保存前比较磁盘文件的修改时间与内容，发现外部修改时要求确认。
- 打开、新建或拖入其他 Markdown 前检查未保存修改。
- 文档内容和文件名使用 `sessionStorage`，每个标签页独立。
- 字号与布局偏好继续使用 `localStorage` 跨标签页共享。
- 关闭标签页时保留浏览器原生未保存提醒。

## 验收标准

- 双击、工具内打开和拖拽 Markdown 后均默认只显示预览区。
- 点击“编辑文档”显示双栏；点击“完成编辑”恢复全宽预览。
- 工具内打开的文件可以在授权后直接保存。
- 双击接管的文件首次保存会显示清晰的源文件授权说明。
- 另存为后后续保存写入新文件；导出 HTML 不清除 Markdown 未保存状态。
- 空文件打开后目录和预览为空，不保留上一文档内容。
- 多标签页之间的内容、文件名和 dirty 状态互不影响。

## 非目标

- 不引入 PWA、Native Messaging 或本地辅助程序。
- 不实现自动保存、版本历史和多文档标签系统。
- 不承诺未经用户授权直接覆盖通过 `file://` 打开的源文件。
