# UX 迭代 Spec：侧栏、目录、字号与标签页标题优化

日期：2026-07-15
目标版本建议：v1.4.0
范围：Chrome 扩展 `md.` 的易用性优化，不改变 Markdown 渲染核心能力。

## 背景

近期使用反馈集中在四个细节：

1. 侧栏工具按钮占用空间过大。
2. 长文档目录一次性展开全部标题，不便定位。
3. 字号档位不足，最大字号仍偏小。
4. 多个 Markdown 文件同时打开时，Chrome 标签页标题都显示 `md.`，不方便区分。

这些问题都属于高频使用路径，且实现点清晰，适合作为一次小版本体验迭代。

## 设计目标

- 保持 GitHub-like 白底黑字风格，不引入复杂视觉装饰。
- 降低侧栏垂直占用，让目录成为侧栏主要内容。
- 长文档目录支持手动折叠/展开，默认状态更利于浏览。
- 字号控制覆盖更大的阅读需求，尤其是投屏、高分屏和视力友好场景。
- Chrome 标签页标题直接反映当前文档名。

## 非目标

- 不重做整体布局。
- 不引入第三方 UI 库。
- 不改 Markdown 解析器和导出格式的核心逻辑。
- 不实现“保存回原文件”，仍保持下载副本的浏览器安全模型。

## 方案 1：侧栏工具按钮压缩

### 当前问题

侧栏顶部 `打开 / 保存 / 导出 / 字号 / 新建` 全部使用纵向块状按钮。默认侧栏宽度有限，按钮区会挤压目录区域，长文档时尤其明显。

### 推荐方案

将工具区改为“紧凑工具条”：

- 第一行：主要动作
  - `打开`
  - `保存`
  - `导出`
- 第二行：辅助动作
  - `字号`
  - `新建`
- 按钮高度从当前约 30px 压缩到约 26px。
- 按钮横向排列，必要时自动换行。
- 保留文字按钮，不改成纯图标，避免增加学习成本。

### 备选方案

折叠为“工具”下拉菜单。
缺点是常用操作多一次点击，且当前项目无菜单组件，复杂度高于收益。

### 验收标准

- 侧栏打开后，工具区高度明显低于当前版本。
- 所有按钮仍可键盘聚焦和点击。
- 小 / 中 / 大 / 更大字号下按钮文字不溢出。

## 方案 2：目录支持折叠/展开

### 当前问题

`buildToc()` 会渲染所有 `h1` 到 `h6` 标题。长文档中标题过多时，目录列表过长，用户难以快速定位当前章节。

### 推荐方案

实现手动折叠树状目录：

- `h1`、`h2` 作为主要节点，显示折叠按钮。
- 默认展开到 `h2`，`h3` 及以下默认折叠。
- 用户可点击折叠按钮展开/收起某个标题分支。
- 当前高亮标题如果在折叠分支内，应自动展开其父级，保证高亮项可见。
- 折叠状态按当前文档维护，可存在内存中；暂不做跨文档持久化，避免旧文档状态污染新文档。

### 数据结构

`buildToc()` 当前直接拼 HTML 字符串。建议增加一步结构化：

```js
[
  { id, text, level, children: [...] }
]
```

渲染时基于树生成 DOM。这样折叠逻辑更清晰，也方便后续增加“全部展开 / 全部折叠”。

### 交互细节

- 有子项的标题前显示一个小型三角按钮。
- 点击标题文字仍执行跳转。
- 点击三角按钮只切换折叠状态，不跳转。
- 当前激活项仍保留 `.active` 高亮。
- 目录自动滚动到当前高亮项的逻辑保留。

### 验收标准

- 长文档中目录不会默认展开所有深层标题。
- 手动展开/折叠不会影响标题跳转。
- 滚动预览区或编辑区时，当前高亮目录项可见。
- 切换新文件后，不沿用上一个文件的折叠状态。

## 方案 3：字号增加更多档位

### 当前问题

当前只有 `small / medium / large` 三档：

- 小：编辑区 13px，预览区 13px
- 中：编辑区 15px，预览区 15px
- 大：编辑区 17px，预览区 17px

反馈中“大号还不够大”。

### 推荐方案

扩展为五档：

| 档位 | UI | 编辑区 | 预览区 | 按钮文案 |
|---|---:|---:|---:|---|
| small | 13px | 13px | 13px | 字号：小 |
| medium | 14px | 15px | 15px | 字号：中 |
| large | 15px | 17px | 17px | 字号：大 |
| xlarge | 16px | 19px | 19px | 字号：特大 |
| xxlarge | 17px | 21px | 21px | 字号：超大 |

字号按钮继续循环切换，保持现有心智模型。

### 注意事项

- 超大字号下侧栏宽度和工具按钮容易拥挤，需要配合工具区压缩。
- 编辑区行高可保持比例值，避免单独为每档维护行高。
- `localStorage` 中旧值 `small / medium / large` 继续兼容。

### 验收标准

- 旧用户字号偏好不丢失。
- 点击字号按钮可循环五档。
- 超大字号下编辑区、预览区、侧栏文本不重叠。

## 方案 4：Chrome 标签页标题显示文件名

### 当前问题

`index.html` 的 `<title>` 固定为 `md.`。打开多个文件时，Chrome 标签页名称相同，不方便区分。

### 推荐方案

在所有更新 `currentFileName` 的路径同步调用：

```js
updateDocumentTitle();
```

标题规则：

- 普通状态：`文件名 - md.`
- 未保存状态：`* 文件名 - md.`
- 未命名文档：`未命名文档.md - md.`

应覆盖以下入口：

- 扩展图标打开并恢复本地内容。
- 工具栏“打开 .md”。
- 双击本地 `.md` 文件经 `content.js` 打开。
- 拖拽 `.md` 文件打开。
- 新建文档。
- 保存为新文件名。
- dirty 状态变化。

### 验收标准

- Chrome 标签页可区分不同 Markdown 文件。
- 文件重命名保存后标签页标题同步更新。
- 未保存修改时标题前有 `*`。
- 保存后 `*` 消失。

## 实施顺序

1. 标签页标题同步。
2. 字号扩展到五档。
3. 侧栏工具区压缩。
4. 目录折叠/展开。

理由：

- 标题同步风险最低，收益直接。
- 字号扩展依赖现有 CSS 变量，改动较小。
- 工具区压缩会影响侧栏视觉，应在字号档位确定后做。
- 目录折叠涉及 `buildToc()` 结构调整，风险最高，最后实施并重点测试。

## 风险与回归点

- 目录折叠可能影响当前目录跳转、高亮和自动滚动逻辑。
- 字号超大档可能导致工具区或侧栏溢出。
- 标签页标题与页面内文件名、dirty 状态需要保持一致。
- 打开空白文件、无标题文档、重复标题文档需要单独验证。

## 测试清单

- 打开多个 `.md` 文件，Chrome 标签页名称分别显示文件名。
- 编辑文件后标签页标题出现 `*`，保存后消失。
- 字号按钮循环五档，刷新后保留上次档位。
- 长文档目录默认折叠深层标题。
- 手动展开/折叠目录后，点击标题仍能跳转。
- 滚动预览区时目录高亮跟随并自动滚入可见区域。
- 打开新文件后目录折叠状态不会沿用旧文件。
- 空白 `.md` 文件打开后目录为空，预览为空，标签页显示文件名。
