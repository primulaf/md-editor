# v1.5.0

发布日期：2026-07-20

v1.5.0 重构阅读与保存体验：文档默认以全宽预览打开，需要修改时再进入编辑模式；保存、另存为和 HTML 导出现在具有清晰且独立的行为。

## 阅读与编辑

- 文件打开后默认进入阅读模式，预览区使用全部可用宽度。
- 新增“编辑文档/完成编辑”模式切换，修改状态在退出编辑模式后继续保留。
- 新建文档会直接进入编辑模式，已有布局比例在重新进入编辑模式后恢复。
- 阅读模式精简侧栏操作，编辑模式再显示保存与另存为。

## 文件保存

- 工具内打开的文件在授权后可直接写回源文件。
- “保存”写入当前 Markdown 目标，“另存为”创建新目标，“导出 HTML”不改变 Markdown 保存状态。
- 双击接管的文件首次保存时显示源文件授权说明，可选择原文件或另存为。
- 保存前检测磁盘文件的外部修改，避免无提示覆盖。
- File System Access API 不可用时自动下载副本，并明确说明源文件未更改。

## 稳定性与细节

- 文档草稿改用标签页级会话存储，多个标签页不再共享内容、文件名和未保存状态。
- 标签页标题和页面标题栏同步显示文件名及未保存标记。
- 空白文件会正确清空目录与预览，不再残留上一文档内容。
- 目录长标题根据侧栏剩余宽度单行省略，悬停仍可查看完整标题。

## 兼容性

- 通过 `file://` 双击接管的文档仍需在第一次写回时选择一次原文件，这是 Chrome 的文件权限限制。
- File System Access API 不可用时自动退回下载副本，并明确提示源文件未更改。

## 验证

- `npm test`、JavaScript 语法检查、Manifest 解析和项目结构校验通过。
- 阅读/编辑切换、源文件保存、另存为、HTML 导出和未保存提醒通过。
- 多标签页隔离、空白文档载入、目录导航和窄屏布局回归通过。

## 安装

下载 Release 附件 `md-editor-extension.zip`，解压后在 `chrome://extensions` 中开启开发者模式并选择“加载已解压的扩展程序”。直接打开本地 Markdown 文件前，请在扩展详情中开启“允许访问文件网址”。
